# Skills

Skills are reusable instruction files that AI agents read as context.
They encode procedural knowledge (how to debug, how to review code, how to plan)
so you don't have to repeat it in every prompt.

## What skills are (and are not)

- **Are**: Markdown files that the agent reads before or during a task
- **Are not**: Plugins, executables, or cloud dependencies
- **Work with**: Aider, Roo Code, Continue, OpenCode, Claude Code — any agent that reads files
- **Source**: https://skills.sh (install with `npx skills add <owner/repo>`)

## Skills included in this stack

Skills live in `skills/<category>/`. They are **not** auto-loaded — you add them
explicitly to the agent context when relevant.

## How to use skills with each orchestrator

### Aider
```bash
# Add a skill as context before your request
aider --read skills/process/systematic-debugging.md

# Or inside a session
/read skills/process/systematic-debugging.md
/ask I'm seeing this error: [error]. Follow the debugging process.
```

### Roo Code (VSCode)
1. Open the skill file in VSCode
2. In the Roo Code chat, use `@` to mention the file:
   `@skills/process/systematic-debugging.md Follow this process to debug: [error]`

### Continue (VSCode)
1. Use `@file` in the Continue chat:
   `@file:skills/process/systematic-debugging.md I have this error: [error]`

### OpenCode
```bash
# Reference the skill file in your prompt
opencode "Read skills/process/test-driven-development.md and apply it to create tests for the items module"
```

## Installing new skills

Skills are plain Markdown files. You can:

1. **Install from skills.sh** (requires Node.js):
   ```bash
   npx skills add obra/superpowers --skill systematic-debugging
   # The SKILL.md file will be placed in the current directory
   # Move it to skills/process/ and rename it
   ```

2. **Write your own**: Create a `.md` file in the appropriate category directory.
   Follow the format of existing skills: overview, when to use, step-by-step process.

3. **Check skills.sh** for the full catalog: https://skills.sh

## Updating skills

Skills from `obra/superpowers` are actively maintained. To update:
```bash
npx skills add obra/superpowers --skill systematic-debugging
# Replace the existing file
```

Check the source repositories for changelogs before updating in a team context.
